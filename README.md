# keio-syllabus-survey-overlay

慶應義塾大学のシラバス・履修登録・K-Support / K-LMS と連動し、授業評価・学生レビュー・改善要望を見やすくするためのブラウザ拡張機能です。

## 背景

慶應には授業評価アンケートの仕組みがありますが、履修登録やシラバス確認のタイミングで、過去の授業評価結果や履修者の実感に自然にアクセスしづらいという課題があります。

特にオンライン授業やグループワークを含む授業では、シラバスだけでは以下のような情報が分かりにくいです。

- 授業が実際に分かりやすいか
- 質問しやすいか
- 学生同士の交流があるか
- グループワークが機能しているか
- 一部の学生に負担が偏っていないか
- オンライン授業として十分に設計されているか
- 教員・TAが授業中の問題に介入してくれるか

このプロジェクトは、学生が授業を選ぶとき、受けているとき、振り返るときに、授業に関する情報をより透明にし、授業改善につなげることを目指します。

## コンセプト

この拡張機能は「教員を晒すためのサービス」ではありません。

目的は、授業がちゃんと機能しているかを学生・教員・大学が早く知り、履修選択と授業改善に活かせるようにすることです。

公式の授業評価結果が存在する場合は、それをログイン済みの本人が見やすい形で参照できるようにします。  
また、学生自身によるレビューや改善要望を集め、次の履修者や授業改善に役立てます。

## 主な機能

### 1. シラバス上で授業評価を表示

シラバスや履修登録画面を開いたときに、その授業に対応する評価情報を表示します。

表示候補:

- 公式授業評価へのリンク
- 学生レビュー
- 履修負荷
- 質問しやすさ
- 授業の分かりやすさ
- グループワークの健全性
- オンライン授業としての設計
- 成績評価の納得感
- 履修者からのアドバイス

### 2. 授業評価を投稿

履修者が授業について評価を投稿できます。

評価対象は、教員個人の人格ではなく、授業設計・運営・学習体験に限定します。

評価項目例:

- 授業内容は理解しやすかったか
- 質問しやすかったか
- 課題量は適切だったか
- 評価基準は明確だったか
- グループワークは機能していたか
- 一部の学生に負担が偏っていなかったか
- オンライン授業としての工夫があったか
- 履修前に知りたかったこと

### 3. 改善要望を匿名で送信

学生が授業中に感じた改善要望を、匿名で送れるようにします。

想定する改善要望:

- グループワークが成立していない
- 欠席者や無言参加者へのルールがない
- 質問先が分からない
- 課題の意図が分かりにくい
- Zoom / Slack / Discord / Gather などの導線が不足している
- オンライン授業なのに、対面授業の単純な置き換えになっている

将来的には、個別の暴言や攻撃ではなく、授業ごとの改善要望サマリーとして教員・TA・大学側に共有できる形を目指します。

## 公式授業評価結果との関係

K-Support には「授業評価結果 科目検索」が存在します。

確認済みの画面:

- URL: `https://keiouniversity.my.site.com/students/s/ClassEvaluationSearch`
- 画面名: 授業評価結果 科目検索 / Search for Class Evaluation Results
- 検索項目:
  - 科目名 Course Name
  - 主担当者 Main Lecturer
  - 学期 Semester
  - キャンパス Campus
  - 学部 Faculty
- 検索結果:
  - 科目名
  - 主担当者
  - 学期
  - 曜日時限
  - キャンパス
  - 学部

この拡張機能では、ユーザーが正規にログイン済みで閲覧できる範囲の公式授業評価結果を、シラバスや履修登録画面から参照しやすくすることを目指します。

## 重要な方針

### やること

- シラバス画面から授業名・教員名・学期などを読み取る
- K-Support上の公式授業評価結果と照合する
- ログイン済みユーザーが閲覧できる公式結果への導線を表示する
- 学生独自のレビューを収集・表示する
- 匿名の改善要望を送れるようにする
- 授業改善に役立つ形で情報を整理する

### やらないこと

- keio.jp / K-Support / K-LMS のIDやパスワードを取得しない
- Cookieや認証トークンを外部送信しない
- 認証回避や権限昇格をしない
- ユーザーが閲覧権限を持たない情報を取得しない
- 公式授業評価結果を外部DBに保存・転載・再公開しない
- 教員個人への誹謗中傷を公開しない
- 個人情報や特定の学生を識別できる情報を扱わない

## データの扱い

### 公式データ

K-Support上の公式授業評価結果は、ログイン済みユーザー本人がその場で閲覧するためだけに扱います。

原則として、公式授業評価結果そのものを外部DBに保存しません。

### 自前データ

以下のような学生投稿データは、自前DBに保存する可能性があります。

- 授業レビュー
- 履修者からのアドバイス
- 改善要望
- グループワークの実態
- オンライン授業の設計に関する評価

ただし、投稿内容は授業設計・学習体験に関するものに限定し、個人攻撃や名誉毀損につながる投稿は扱いません。

## 想定するUI

シラバスや履修登録画面に、以下のようなカードを追加します。

- 公式授業評価を見る
- この授業の学生レビューを見る
- この授業を評価する
- 改善要望を匿名で送る

例:

- 総合評価: 4.2 / 5
- 履修負荷: 高め
- 質問しやすさ: 3.8 / 5
- グループワークの健全性: 2.1 / 5
- オンライン授業設計: 2.5 / 5
- 履修者コメント: 「課題量は多いが、TAのサポートは手厚い」

## 技術構成案

### Browser Extension

- Chrome Extension
- Safari Web Extension 対応も将来的に検討
- Manifest V3
- Content Script
- Background Service Worker

### Backend

候補:

- Firebase
- Supabase
- Cloudflare Workers + D1

扱うデータ:

- 自前の学生レビュー
- 改善要望
- 授業IDとの対応情報
- 公開可能な集計情報

### Extension Flow

1. ユーザーがシラバスまたは履修登録画面を開く
2. Content Script が授業名・教員名・学期などをDOMから取得する
3. K-Supportの授業評価結果検索と照合する
4. 公式結果がある場合、結果ページへの導線を表示する
5. 自前DBから学生レビューを取得する
6. シラバス画面上に評価カードを表示する

## 授業の照合キー

優先的に利用する情報:

- 科目名
- 主担当者
- 学期
- キャンパス
- 学部
- 曜日時限

完全一致が難しい場合は、複数条件を組み合わせて候補を出し、ユーザーに確認させます。

## 倫理・モデレーション方針

このプロジェクトは、授業改善と履修選択の支援を目的とします。

以下の投稿は禁止します。

- 教員・学生への誹謗中傷
- 人格攻撃
- 容姿・国籍・性別・属性への言及
- 個人情報
- 事実確認が困難な告発
- 授業と無関係な悪口
- 他人を特定できる情報

自由記述は、公開前にモデレーションする方針です。
